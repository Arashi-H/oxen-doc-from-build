\doxysubsection{Detailed Description}
Service node registration details.

A registration contains any reserved amounts + addresses starting with the operator, and must always have at least one (the operator).

Before H\+F19 this is limited to 4 spots per registrations, and amounts/fees are specified in portions (i.\+e. the numerator of a fraction with denominator 2$^\wedge$64 -\/ 4).

Starting in H\+F19 we allow 10 spots per registration, reserved amounts are specified in atomic O\+X\+EN (i.\+e. 1000000000 = 1 O\+X\+EN), and the operator fee is specified in a per-\/million value (i.\+e. 1000000 = 100\%).

The {\ttfamily hf\+\_\+or\+\_\+expiration} field, before H\+F19, is a timestamp at which the registration expires; starting in H\+F19, it is instead a hardfork version of the registration which must agree with the current hardfork version (e.\+g. if set to 23 then the registration is only accepted on H\+F23).

For crossing-\/the-\/hardfork compatibility, we allow pre-\/\+H\+F19 registrations (i.\+e. using a timestamp + portions + at most 4 reserved spots) during H\+F19 (but not in H\+F20+).
