\doxysubsection{Detailed Description}
Full block information can be retrieved by either block height or hash, like with the above block header calls. For full block information, both lookups use the same method, but with different input parameters.

Inputs\+:


\begin{DoxyItemize}
\item {\ttfamily hash} The block\textquotesingle{}s hash.
\item {\ttfamily height} A block height to look up; returned in {\ttfamily \mbox{\hyperlink{structcryptonote_1_1block__header}{block\+\_\+header}}}
\item {\ttfamily fill\+\_\+pow\+\_\+hash} Tell the daemon if it should fill out pow\+\_\+hash field.
\end{DoxyItemize}

Output values available from a public R\+PC endpoint\+:


\begin{DoxyItemize}
\item {\ttfamily status} General R\+PC status string. {\ttfamily \char`\"{}\+O\+K\char`\"{}} means everything looks good.
\item {\ttfamily \mbox{\hyperlink{structcryptonote_1_1block__header}{block\+\_\+header}}} Block header information for the requested {\ttfamily height} block
\item {\ttfamily tx\+\_\+hashes} List of hashes of non-\/coinbase transactions in the block. If there are no other transactions, this will be an empty list.
\item {\ttfamily blob} Hexadecimal blob of block information.
\item {\ttfamily json} J\+S\+ON formatted block details. 
\end{DoxyItemize}
