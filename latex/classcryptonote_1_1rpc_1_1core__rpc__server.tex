\doxysubsection{Detailed Description}
Core R\+PC server.

This class handles all internal core R\+PC requests, but does not itself listen for anything external. It is meant to be used by other R\+PC server bridge classes (such as \mbox{\hyperlink{classcryptonote_1_1rpc_1_1http__server}{rpc\+::http\+\_\+server}}) to map incoming H\+T\+TP requests into internal core R\+PC requests through this class, and then send them back to the requester.

In order to add a new R\+PC request object you must\+:


\begin{DoxyItemize}
\item add the appropriate N\+E\+W\+T\+Y\+PE struct with request/response substructs to \mbox{\hyperlink{core__rpc__server__commands__defs_8h}{core\+\_\+rpc\+\_\+server\+\_\+commands\+\_\+defs.\+h}}; the base types it inherits from determine the permissions and data type, and a static {\ttfamily names()} method determined the rpc name (and any older aliases).
\item add an \mbox{\hyperlink{classcryptonote_1_1rpc_1_1core__rpc__server_a1ca4989e4ab5bb27dde2172000152fa5}{invoke()}} method overload declaration here which takes a N\+E\+W\+T\+Y\+P\+E\+::request and \mbox{\hyperlink{structcryptonote_1_1rpc_1_1rpc__context}{rpc\+\_\+context}}, and returns a N\+E\+W\+T\+Y\+P\+E\+::response.
\item add the \mbox{\hyperlink{classcryptonote_1_1rpc_1_1core__rpc__server_a1ca4989e4ab5bb27dde2172000152fa5}{invoke()}} definition in \mbox{\hyperlink{core__rpc__server_8cpp}{core\+\_\+rpc\+\_\+server.\+cpp}}, and add N\+E\+W\+T\+Y\+PE to the list of command types near the top of \mbox{\hyperlink{core__rpc__server_8cpp}{core\+\_\+rpc\+\_\+server.\+cpp}}. 
\end{DoxyItemize}
