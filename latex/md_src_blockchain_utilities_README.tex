{\ttfamily \$ oxen-\/blockchain-\/export}

This loads the existing blockchain and exports it to {\ttfamily \$\+O\+X\+E\+N\+\_\+\+D\+A\+T\+A\+\_\+\+D\+IR/export/blockchain.raw}\hypertarget{md_src_blockchain_utilities_README_autotoc_md4}{}\doxysubsection{Import the exported file}\label{md_src_blockchain_utilities_README_autotoc_md4}
{\ttfamily \$ oxen-\/blockchain-\/import}

This imports blocks from {\ttfamily \$\+O\+X\+E\+N\+\_\+\+D\+A\+T\+A\+\_\+\+D\+IR/export/blockchain.raw} (exported using the {\ttfamily oxen-\/blockchain-\/export} tool as described above) into the current database.

Defaults\+: {\ttfamily -\/-\/batch on}, {\ttfamily -\/-\/batch size 20000}, {\ttfamily -\/-\/verify on}

Batch size refers to number of blocks and can be adjusted for performance based on available R\+AM.

Verification should only be turned off if importing from a trusted blockchain.

If you encounter an error like \char`\"{}resizing not supported in batch mode\char`\"{}, you can just re-\/run the {\ttfamily oxen-\/blockchain-\/import} command again, and it will restart from where it left off.

{\ttfamily -\/-\/database $<$database type$>$}

{\ttfamily -\/-\/database $<$database type$>$\#$<$flag(s)$>$}

flags\+:

The flag after the \# is interpreted as a composite mode/flag if there\textquotesingle{}s only one (no comma separated arguments).

The composite mode represents multiple DB flags and support different database types\+:

{\ttfamily safe, fast, fastest}

Database-\/specific flags can be set instead.
