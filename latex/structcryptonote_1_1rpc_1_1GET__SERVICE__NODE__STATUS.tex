\doxysubsection{Detailed Description}
Retrieves information on the current daemon\textquotesingle{}s Service Node state. The returned information is the same as what would be returned by \char`\"{}get\+\_\+service\+\_\+nodes\char`\"{} when passed this service node\textquotesingle{}s public key.

Inputs\+: none.

Outputs\+:
\begin{DoxyItemize}
\item {\ttfamily service\+\_\+node\+\_\+state} -\/ if this is a registered service node then all available fields for this service node. \begin{DoxySeeAlso}{See also}
\mbox{\hyperlink{structcryptonote_1_1rpc_1_1GET__SERVICE__NODES}{G\+E\+T\+\_\+\+S\+E\+R\+V\+I\+C\+E\+\_\+\+N\+O\+D\+ES}} \mbox{\hyperlink{ommentedCombined_2pow225521_8h_a67d1613adb06cf7e2b1da5a5f82c5900}{for}} the list of fields. Note that some fields (such as remote testing results) will not be available (through this call or {\ttfamily \char`\"{}get\+\_\+service\+\_\+nodes\char`\"{}}) because a service node is incapable of testing itself \mbox{\hyperlink{ommentedCombined_2pow225521_8h_a67d1613adb06cf7e2b1da5a5f82c5900}{for}} remote connectivity. If this daemon is running in service node mode but not registered then only SN pubkey, ip, and port fields are returned.
\end{DoxySeeAlso}

\item {\ttfamily height} current top block height at the time of the request (note that this is generally one less than the \char`\"{}blockchain height\char`\"{}).
\item {\ttfamily block\+\_\+hash} current top block hash at the time of the request
\item {\ttfamily status} generic R\+PC error code; \char`\"{}\+O\+K\char`\"{} means the request was successful. 
\end{DoxyItemize}
