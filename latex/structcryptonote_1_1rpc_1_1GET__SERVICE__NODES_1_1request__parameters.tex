Set of fields to return; listed fields apply to both the top level (such as {\ttfamily \char`\"{}height\char`\"{}} or {\ttfamily \char`\"{}block\+\_\+hash\char`\"{}}) and to keys inside {\ttfamily service\+\_\+node\+\_\+states}. Fields should be provided as a list of field names to include. For backwards compatibility when making a json request field names can also be provided as a dictionary of \{\char`\"{}field\+\_\+name\char`\"{}\+: true\} pairs, but this usage is deprecated (and not supported for bt-\/encoded requests).

The special field name \char`\"{}all\char`\"{} can be used to request all available fields; this is the default when no fields key are provided at all. Be careful when requesting all fields\+: the response can be very large.

When providing a list you may prefix a field name with a {\ttfamily -\/} to remove the field from the list; this is mainly useful when following \char`\"{}all\char`\"{} to remove some fields from the returned results.
