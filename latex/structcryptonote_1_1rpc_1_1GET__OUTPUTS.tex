\doxysubsection{Detailed Description}
Retrieve transaction outputs

Inputs\+:


\begin{DoxyItemize}
\item {\ttfamily outputs} Array of output indices. For backwards compatibility these may also be passed as an array of \{\char`\"{}amount\char`\"{}\+:0,\char`\"{}index\char`\"{}\+:n\} dicts.
\item {\ttfamily get\+\_\+txid} Request the T\+X\+ID (i.\+e. hash) of the transaction as well.
\item {\ttfamily as\+\_\+tuple} Requests the returned outs variable as a tuple of values rather than a dict.
\end{DoxyItemize}

Output values available from a public R\+PC endpoint\+:


\begin{DoxyItemize}
\item {\ttfamily status} General R\+PC status string. {\ttfamily \char`\"{}\+O\+K\char`\"{}} means everything looks good.
\item {\ttfamily outs} List of outkey information; if {\ttfamily as\+\_\+tuple} is not set then these are dicts containing keys\+:
\begin{DoxyItemize}
\item {\ttfamily key} The public key of the output.
\item {\ttfamily mask} 
\item {\ttfamily unlocked} States if output is locked ({\ttfamily false}) or not ({\ttfamily true}).
\item {\ttfamily height} Block height of the output.
\item {\ttfamily txid} Transaction id; only present if requested via the {\ttfamily get\+\_\+txid} parameter. Otherwise, when {\ttfamily as\+\_\+tuple} is set, these are 4-\/ or 5-\/element arrays (depending on whether {\ttfamily get\+\_\+txid} is desired) containing the values in the order listed above. 
\end{DoxyItemize}
\end{DoxyItemize}
