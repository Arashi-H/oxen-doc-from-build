\doxysubsection{Detailed Description}
Base class that all R\+PC commands must inherit from (either directly or via one or more of the below tags). Inheriting from this (and no others) gives you a private, json, non-\/legacy R\+PC command. For L\+MQ R\+PC the command will be available at {\ttfamily admin.\+whatever}; for H\+T\+TP R\+PC it\textquotesingle{}ll be at {\ttfamily whatever}. This base class is also where response objects are stored.

The response data. For bt-\/encoded responses we convert this on the fly, with the following notes\+:
\begin{DoxyItemize}
\item boolean values become 0 or 1
\item key-\/value pairs with null values are omitted from the object
\item other null values are not permitted at all\+: an exception will be raised if the json contains such a value.
\item double values are not permitted; if a double is absolutely needed then check {\ttfamily is\+\_\+bt} and, when bt, encode it in some documented, endpoint-\/specific way.
\item binary values in strings {\itshape are} permitted, but the caller must take care because they will not be permitted for actual json responses (json serialization will fail)\+: the caller is expected to do something like\+: \begin{DoxyVerb}std::string binary = some_binary_data();
cmd.response["binary_value"] = is_bt ? binary : oxenmq::to_hex(binary);
\end{DoxyVerb}

\end{DoxyItemize}

or, more conveniently, using the shortcut interface\+: \begin{DoxyVerb}cmd.response_hex["binary_value"] = some_binary_data();
