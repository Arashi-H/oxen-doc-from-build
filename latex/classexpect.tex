\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T$>$\newline
class expect$<$ T $>$}

{\ttfamily expect$<$T$>$} is a value or error implementation, similar to Rust std\+::result or various C++ proposals (boost\+::expected, boost\+::outcome). This implementation currently has a strict error type, {\ttfamily std\+::error\+\_\+code}, and a templated value type {\ttfamily T}. {\ttfamily expect$<$T$>$} is implicitly convertible from {\ttfamily T} or {\ttfamily std\+::error\+\_\+code}, and one {\ttfamily expect$<$T$>$} object type is implicitly convertible to another {\ttfamily expect$<$U$>$} object iff the destination value type can be implicitly constructed from the source value type (i.\+e. {\ttfamily struct U \{ ... U(\+T src) \{ ...\} ... \};}).

{\ttfamily operator==} and {\ttfamily operator!=} are the only comparison operators provided; comparison between different value types is allowed provided the two values types have a {\ttfamily operator==} defined between them (i.\+e. {\ttfamily assert(expect$<$int$>$\{100\} == expect$<$short$>$\{100\});}). Comparisons can also be done against {\ttfamily std\+::error\+\_\+code} objects or error code enums directly (i.\+e. {\ttfamily assert(expect$<$int$>$\{make\+\_\+error\+\_\+code(common\+\_\+error\+::k\+Invalid\+Argument)\} == error\+::k\+Invalid\+Argument)}). Comparison of default constructed {\ttfamily std\+::error\+\_\+code} will always fail. \char`\"{}\+Generic\char`\"{} comparisons can be done with {\ttfamily std\+::error\+\_\+condition} via the {\ttfamily matches} method only (i.\+e. {\ttfamily assert(expect$<$int$>$\{make\+\_\+error\+\_\+code\{\mbox{\hyperlink{common_2error_8h_aa7064892449ea8f8d9b9832971915f29ae3220191425298c4c5ec57290132820e}{common\+\_\+error\+::k\+Invalid\+Error\+Code}}\}.matches(std\+::errc\+::invalid\+\_\+argument))}), {\ttfamily operator==} and {\ttfamily operator!=} will not work with {\ttfamily std\+::errc} or {\ttfamily std\+::error\+\_\+condition}. A comparison with {\ttfamily matches} is more expensive because an equivalency between error categories is computed, but is recommended when an error can be one of several categories (this is going to be the case in nearly every situation when calling a function from another C++ struct/class).

{\ttfamily expect$<$void$>$} is a special case with no stored value. It is used by functions that can fail, but otherwise would return {\ttfamily void}. It is useful for consistency; all macros, standalone functions, and comparison operators work with {\ttfamily expect$<$void$>$}.

\begin{DoxyNote}{Note}
See {\ttfamily \mbox{\hyperlink{common_2error_8h}{src/common/error.\+h}}} for creating a custom error enum. 
\end{DoxyNote}
