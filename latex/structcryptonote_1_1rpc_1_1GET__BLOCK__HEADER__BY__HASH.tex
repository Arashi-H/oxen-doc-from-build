\doxysubsection{Detailed Description}
Block header information can be retrieved using either a block\textquotesingle{}s hash or height. This method includes a block\textquotesingle{}s hash as an input parameter to retrieve basic information about the block.

Inputs\+:
\begin{DoxyItemize}
\item {\ttfamily hash} The block\textquotesingle{}s S\+H\+A256 hash.
\item {\ttfamily hashes} Request multiple blocks via an array of hashes
\item {\ttfamily fill\+\_\+pow\+\_\+hash} Tell the daemon if it should fill out pow\+\_\+hash field.
\item {\ttfamily get\+\_\+tx\+\_\+hashes} If true (default false) then include the hashes of non-\/coinbase transactions
\end{DoxyItemize}

Output values available from a public R\+PC endpoint\+:


\begin{DoxyItemize}
\item {\ttfamily status} General R\+PC status string. {\ttfamily \char`\"{}\+O\+K\char`\"{}} means everything looks good.
\item {\ttfamily \mbox{\hyperlink{structcryptonote_1_1block__header}{block\+\_\+header}}} Block header information for the requested {\ttfamily hash} block
\item {\ttfamily block\+\_\+headers} Block header information for the requested {\ttfamily hashes} blocks 
\end{DoxyItemize}
