\doxysubsection{Detailed Description}
Generate the signature necessary for updating the requested record using the wallet\textquotesingle{}s active \mbox{[}sub\mbox{]}address\textquotesingle{}s spend key. The signature is only valid if the queried wallet is one of the owners of the O\+NS record.

This command is only required if the open wallet is one of the owners of a O\+NS record but wants the update transaction to occur via another non-\/owning wallet. By default, if no signature is specified to the update transaction, the open wallet is assumed the owner and it\textquotesingle{}s active \mbox{[}sub\mbox{]}address\textquotesingle{}s spend key will automatically be used.

Inputs\+:


\begin{DoxyItemize}
\item {\ttfamily type} -- The mapping type, currently we support \char`\"{}session\char`\"{}, \char`\"{}lokinet\char`\"{} and \char`\"{}wallet\char`\"{} mappings.
\item {\ttfamily name} -- The desired name to update via Oxen Name Service
\item {\ttfamily encrypted\+\_\+value} -- (Optional)\+: The new encrypted value that the name maps to via Oxen Name Service. If not specified or given the empty string \char`\"{}\char`\"{}, then the mapping\textquotesingle{}s value remains unchanged.
\item {\ttfamily owner} -- (Optional)\+: The new owner of the mapping. If not specified or given the empty string \char`\"{}\char`\"{}, then the mapping\textquotesingle{}s owner remains unchanged.
\item {\ttfamily backup\+\_\+owner} -- (Optional)\+: The new backup owner of the mapping. If not specified or given the empty string \char`\"{}\char`\"{}, then the mapping\textquotesingle{}s backup owner remains unchanged.
\item {\ttfamily account\+\_\+index} -- (Optional) Use this wallet\textquotesingle{}s subaddress account for generating the signature
\end{DoxyItemize}

Outputs\+:


\begin{DoxyItemize}
\item {\ttfamily signature} -- A signature valid for using in O\+NS to update an underlying mapping. 
\end{DoxyItemize}
