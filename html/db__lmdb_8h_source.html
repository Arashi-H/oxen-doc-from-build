161 // If m_batch_active is set, a batch transaction exists beyond this class, such
162 // as a batch import with verification enabled, or possibly (later) a batch
163 // network sync.
164 //
165 // For some of the lookup methods, such as get_block_timestamp(), tx_exists(),
166 // and get_tx(), when m_batch_active is set, the lookup uses the batch
167 // transaction. This isn't only because the transaction is available, but it's
168 // necessary so that lookups include the database updates only present in the
169 // current batch write.
170 //
171 // A regular network sync without batch writes is expected to open a new read
172 // transaction, as those lookups are part of the validation done prior to the
173 // write for block and tx data, so no write transaction is open at the time.
