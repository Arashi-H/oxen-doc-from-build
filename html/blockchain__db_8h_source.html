1825  // This function accepts an empty timestamps/difficulties array to fill, or
1826  // a prior timestamps/difficulties array that was filled by a previous call to
1827  // this same function in which case it will optimally insert and remove the
1828  // new data instead of reconstructing the entrie array.
1829  //
1830  // timestamps: On return, timestamps of the last loaded
1831  // 'DIFFICULTY_WINDOW' blocks starting from top_block_height stored in
1832  // ascending block height order.
1833  //
1834  // difficulties: On return, cumulative difficulties of the last loaded
1835  // DIFFICULTY_WINDOW blocks starting from 'top_block_height' stored in
1836  // ascending block height order
1837  //
1838  // chain_height: The blockchain height, (next height a block will be added
1839  // at). The input arrays will be filled with the prior `timestamps` and
1840  // `difficulties` DIFFICULTY_WINDOW worth of values.
1841  //
