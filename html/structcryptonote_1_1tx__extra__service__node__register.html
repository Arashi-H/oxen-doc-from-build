Detailed Description

Service node registration details.

A registration contains any reserved amounts + addresses starting with the operator, and must always have at least one (the operator).

Before HF19 this is limited to 4 spots per registrations, and amounts/fees are specified in portions (i.e. the numerator of a fraction with denominator 2^64 - 4).

Starting in HF19 we allow 10 spots per registration, reserved amounts are specified in atomic OXEN (i.e. 1000000000 = 1 OXEN), and the operator fee is specified in a per-million value (i.e. 1000000 = 100%).

The hf_or_expiration field, before HF19, is a timestamp at which the registration expires; starting in HF19, it is instead a hardfork version of the registration which must agree with the current hardfork version (e.g. if set to 23 then the registration is only accepted on HF23).

For crossing-the-hardfork compatibility, we allow pre-HF19 registrations (i.e. using a timestamp + portions + at most 4 reserved spots) during HF19 (but not in HF20+).
