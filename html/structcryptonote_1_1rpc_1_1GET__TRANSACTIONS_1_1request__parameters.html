◆ data

bool cryptonote::rpc::GET_TRANSACTIONS::request_parameters::data = true

Controls whether the data (or pruned, if pruned) field containing raw tx data is included: if explicitly specified then the raw data will be included if true. Otherwise the raw data is included only when neither of split nor prune are set to true.

◆ memory_pool

bool cryptonote::rpc::GET_TRANSACTIONS::request_parameters::memory_pool = false

If true then return all transactions and spent key images currently in the memory pool. This field is exclusive of tx_hashes.

◆ prune

bool cryptonote::rpc::GET_TRANSACTIONS::request_parameters::prune = false

Like split, but also omits the prunable part of transactions from the response details.

◆ split

bool cryptonote::rpc::GET_TRANSACTIONS::request_parameters::split = false

If set to true then always split transactions into non-prunable and prunable parts in the response.

◆ tx_extra

bool cryptonote::rpc::GET_TRANSACTIONS::request_parameters::tx_extra = false

If set to true then parse and return tx-extra information.

◆ tx_hashes

std::vector<crypto::hash> cryptonote::rpc::GET_TRANSACTIONS::request_parameters::tx_hashes

List of transaction hashes to look up. (Will also be accepted as json input key "txs_hashes" for backwards compatibility). Exclusive of memory_pool.
