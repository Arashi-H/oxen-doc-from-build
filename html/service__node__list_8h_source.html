542  // Called via RPC from storage server/lokinet to report a ping test result for a remote storage
543  // server/lokinet.
544  //
545  // How this works:
546  // - SS randomly picks probably-good nodes to test every 10s (with fuzz), and pings
547  // known-failing nodes to re-test them.
548  // - SS re-tests nodes with a linear backoff: 10s+fuzz after the first failure, then 20s+fuzz,
549  // then 30s+fuzz, etc. (up to ~2min retest intervals)
550  // - Whenever SS gets *any* ping result at all it notifies us via RPC (which lands here), and it
551  // is (as of 9.x) our responsibility to decide when too many bad pings should be penalized.
552  //
553  // Our rules are as follows:
554  // - if we have received only failures for more than 1h5min *and* we have at least one failure
555  // in the last 10min then we consider SS reachability to be failing.
556  // - otherwise we consider it good. (Which means either it passed a reachability test at least
557  // once in the last 1h5min *or* SS stopped pinging it, perhaps because it restarted).
558  //

562  // We do all this by tracking three values:
563  // - last_reachable
564  // - first_unreachable
565  // - last_unreachable

567  // On a good ping, we set last_reachable to the current time and clear first_unreachable. On a
568  // bad ping we set last_unreachable to the current time and, if first_unreachable is empty, set
569  // it to current time as well.
570  //
571  // This then lets us figure out:
572  // - current status can be good (first_unreachable == 0), passable (last_unreachable < 10min ago), or failing.
573  // - current *failing* status (current status == failing && first_unreachable more than 1h5min ago)
574  // - last test time (max(last_reachable, last_unreachable), "not yet" if this is 0)

576  // - how long it has been unreachable (now - first_unreachable, if first_unreachable is set)
